redux-thunk

1.redux的概念及原理

在组件内创建一个 action变量 通过 store.dispatch(action)传给 store仓库管理 但是 store不知道 如何处理这个状态 只能通过reducer来处理 嗯 此时的store接收到了传递的action 它有两个参数 一个是初始的状态 另一个是action 的 type 当 dispacth的时候 它的这两个参数都会被reducer·接受到 故而state·的处理在 reduce内 reduce·内不能直接处理state 但可以重新定义一个新的state来获取改变的state 将处理过的state重新返回给store 然后 store再返回给组件。

首先 redux 使用 createStore
然后在组件内分发状态 action 通过store.dispatch(action)触发
此时 状态告知给store store不知如何处理 通过 reducer 查询 与初始定义的状态做比较 是否发生变化
然后再将对比的结果告知store store再传给组件state的状态是否更改了

store.getState()是获取到 redux的所有状态
store.subscribe()可以订阅store的改变 只要store发生改变 该函数就会执行 store.subscribe(this.handleStoreChange)
